## Capability Inventory

For each capability: functional description, bounded context(s), current maturity, strategic priority.

| # | Capability | Description | Bounded Context(s) | Current Maturity | Strategic Priority |
| --- | --- | --- | --- | --- | --- |
| CAP-01 | **Ingest heterogeneous data** | Connect to and ingest data from bureaus (Serasa, Equifax, PH3A), Open Finance, CRMs, ERPs, and custom APIs | BC8 (Connectors) | 🟢 Functional — Liquid Pass has working bureau integrations | P1 — Foundation |
| CAP-02 | **Standardize and reconcile data** | Transform raw multi-source data into consistent, queryable formats with quality rules | BC8 (Connectors), BC2 (Entity) | 🟡 Prototype — Pass v2 has decoupled providers | P1 — Foundation |
| CAP-03 | **Resolve and deduplicate entities** | Ensure same CPF/CNPJ = same entity across the entire system, regardless of tenant | BC2 (Entity) | 🔴 Nonexistent — Critical architecture piece not yet built | P1 — Foundation |
| CAP-04 | **Enrich with external sources** | Augment entity/asset data with market intelligence, news, regulatory filings, monitoring data | BC3 (Market Intel), BC8 (Connectors) | 🔴 Nonexistent — Monitoring Agents not built | P2 — Intelligence |
| CAP-05 | **Calculate risk** | Produce credit risk scores, fraud indicators, and risk assessments using policies and agent analysis | BC7 (Policy), BC6 (Agents) | 🟢 Functional — Liquid Pass v1/v2 | P1 — Foundation |
| CAP-06 | **Estimate liquidity** | Assess an asset’s liquidity potential, pricing, and market fit | BC4 (Liquidity), BC6 (Agents) | 🟡 Prototype — Transação Carteira has early capabilities | P2 — Intelligence |
| CAP-07 | **Detect events and anomalies** | Identify changes, triggers, and anomalies in monitored entities or portfolios | BC3 (Market Intel), BC4 (Liquidity) | 🔴 Nonexistent | P2 — Intelligence |
| CAP-08 | **Prioritize actions** | Rank and recommend next actions for users based on urgency, impact, and opportunity | BC6 (Agents), BC5 (Tasks) | 🔴 Nonexistent | P2 — Intelligence |
| CAP-09 | **Narrate status in natural language** | Generate human-readable summaries, explanations, and narratives about entities, decisions, and processes | BC6 (Agents) | 🔴 Nonexistent — LLM capability exists but not integrated | P1 — Foundation |
| CAP-10 | **Orchestrate agents** | Route tasks to appropriate agents, manage handoffs, coordinate multi-agent workflows | BC6 (Agent Orchestration) | 🔴 Nonexistent — Architecture designed but not implemented | P1 — Foundation |
| CAP-11 | **Execute deterministic policies** | Run Policy-as-Code rules that produce deterministic, auditable outputs unbreakable by agents | BC7 (Policy) | 🟢 Functional — Liquid Pass v2 decision graphs | P1 — Foundation |
| CAP-12 | **Operate human+agent workflows** | Manage processes where humans and agents collaborate (Channels, repasse operations, liquidity operations) | BC5 (Tasks), BC4 (Liquidity), BC6 (Agents) | 🟡 Prototype — Soma operations are human-only; concept validated | P1 — Foundation |
| CAP-13 | **Generate evidence and audit trails** | Create immutable, auditable records of every decision, action, and data access | BC11 (Governance) | 🟡 Prototype — Liquid Pass has basic audit; not system-wide | P1 — Foundation |
| CAP-14 | **Bill usage** | Track, calculate, and bill for SaaS subscriptions, per-action agent usage, and liquidity take rates | BC9 (Billing) | 🔴 Nonexistent — for new platform; existing products have basic billing | P2 — Scale |
| CAP-15 | **Manage repasse lifecycle** | End-to-end agent-assisted mortgage transfer from qualification to settlement | BC4 (Liquidity), BC6 (Agents) | 🟡 Prototype — Soma does this manually; process knowledge validated | P1 — Wedge |
| CAP-16 | **Create and serve dashboards** | Generate, persist, share, and publish interactive data dashboards via agent or manual configuration | BC3 (Market Intel), BC5 (Tasks) | 🟡 Prototype — Gil built Data Blast prototype | P2 — Intelligence |
| CAP-17 | **Execute external outreach (Orbital)** | Send agents to external platforms (WhatsApp, email) to collect information from third parties and bring it back | BC6 (Agents), BC12 (Notifications) | 🔴 Nonexistent — Concept only | P2 — Differentiation |
| CAP-18 | **Facilitate liquidity transactions** | Match, structure, intermediate, or operate financial transactions on real estate assets | BC4 (Liquidity) | 🟡 Prototype — Transação Carteira validates | P3 — Liquidity |
| CAP-19 | **Self-serve onboarding** | Allow new users to discover, register, and start using the platform without human sales contact | BC10 (Growth), BC1 (Identity) | 🔴 Nonexistent | P2 — Growth |
| CAP-20 | **Continuous portfolio monitoring** | Agent-powered ongoing tracking of credit portfolio health, anomalies, and risk changes | BC3 (Market Intel), BC6 (Agents) | 🟢 Functional — Liquid Play (basic) | P2 — Intelligence |
