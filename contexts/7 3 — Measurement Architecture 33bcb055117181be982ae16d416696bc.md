## 1. North Star Metric

**Capital facilitated through the platform** (R$ value of liquidity transactions originated, intermediated, structured, or operated by Liquid)

This is the single metric that captures whether the entire system is working: from community → intelligence → operations → liquidity.

---

## 2. Metric Layers

### Input Metrics (what we put in)

| Metric | Target | Measurement |
| --- | --- | --- |
| Entity Agents active | 50 → 500 → 5,000 | Count of entities monitored |

| Content published (SEO) | 70+ reports, growing | Pages indexed by Google |
| Team capacity | Track | Engineering hours available |

### Process Metrics (how the system operates)

| Metric | Target | Measurement |
| --- | --- | --- |
| Agent task completion rate | > 90% | Tasks completed / tasks created |
| Policy compliance rate | 100% | Policy evaluations followed / total evaluations |
| Average response time | < 30 seconds | Time from task creation to first agent response |
| Token cost per task | Optimize continuously | Average token consumption by task type |
| Bureau query cost per analysis | Track and optimize | Cost per credit analysis operation |
| Kernel scoring accuracy | Calibrate with outcomes | Predicted vs actual liquidity outcomes |

## 3. Agent-Specific Quality Metrics

| Metric | What It Measures | Target | Alert Threshold |
| --- | --- | --- | --- |
| Human rejection rate | % of agent outputs rejected | < 10% | > 20% triggers review |
| Self-correction rate | % of failures agent fixes alone | > 50% | < 30% triggers retraining |
| Orbital Agent success rate | % of external outreach returning valid data | > 70% | < 50% triggers channel review |
| Kernel scoring calibration | Predicted vs actual outcomes | Continuous improvement | Significant deviation triggers recalibration |
| Cost per agent action | Token + bureau + infra cost | Below revenue per action | Above revenue = unit economics broken |
