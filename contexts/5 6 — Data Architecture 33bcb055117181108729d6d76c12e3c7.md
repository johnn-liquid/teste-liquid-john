## 2. Data Quality Rules

| Rule | Application | Enforcement |
| --- | --- | --- |
| **Completeness check** | Every entity must have minimum required fields before scoring | Kernel circuit breaker — incomplete data = cannot score |
| **Format standardization** | Interest rates, dates, currency values must follow canonical format | Data ingestion pipeline normalizes; rejects non-conforming |
| **Freshness** | Bureau data must be < 30 days for credit decisions; market data < 7 days for kernel | Connector metadata tracks last_updated; stale data flagged |
| **Deduplication** | Entity resolution ensures no duplicate CPF/CNPJ | BC2 entity resolution as mandatory gateway |
| **Verification** | Liquid-verified data tagged differently from unverified public data | Reliability hierarchy in agent context |

## 4. International Data Adapters

The kernel is universal; data sources vary by jurisdiction:

| Jurisdiction | Key Sources | Access | Gap |
| --- | --- | --- | --- |
| **USA** | FRED (free), SEC EDGAR (free), Zillow Research (partial free), SIFMA ($5-20k/yr), CoStar ($50k+) | Data is commodity | Contextualization is the value, not access |
| **EU** | ECB data, national registries, ESMA | Varies by country | Fragmentation |
| **LATAM** | Central bank data, local registries | Limited availability | Infrastructure gaps |

**Architectural principle:** Data is commodity, kernel is proprietary. Same kernel, different adapters per jurisdiction.
