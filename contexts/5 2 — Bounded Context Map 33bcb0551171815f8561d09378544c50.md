## 2. Context Types

- **Core:** Contains the primary business logic that differentiates Liquid. These contexts receive the most investment and attention. (BC2, BC3, BC4, BC6, BC7)
- **Supporting:** Necessary for the system but doesn’t contain unique competitive advantage. Could potentially be partially outsourced or use off-the-shelf components. (BC5, BC8, BC9, BC10)
- **Generic:** Common infrastructure shared by most applications. Standard solutions preferred. (BC1, BC11, BC12)

## 3. Context Relationships

### Key Relationships

| From | To | Relationship Type | What Flows |
| --- | --- | --- | --- |
| BC6 (Agent Orchestration) | BC5 (Tasks) | Customer-Supplier | Agents consume and update Tasks; Orchestrator routes Tasks to Agents |
| BC6 (Agent Orchestration) | BC7 (Policy Engine) | Partnership | Agents call Policies for deterministic decisions; Policies return results |
| BC6 (Agent Orchestration) | BC8 (Connectors) | Customer-Supplier | Agents request data from Connectors; Connectors return structured data |
| BC4 (Liquidity Ops) | BC5 (Tasks) | Partnership | Liquidity Trigger creates Channel de Liquidez (special Task/Channel); Liquidity operations run within Channels |
| BC4 (Liquidity Ops) | BC2 (Entity Mgmt) | Customer-Supplier | Liquidity operations reference entities (assets, contracts, parties); Entity context provides verified entity data |
| BC4 (Liquidity Ops) | BC9 (Billing) | Customer-Supplier | Completed liquidity transactions emit billing events; Billing calculates take rate |
| BC3 (Market Intelligence) | BC2 (Entity Mgmt) | Partnership | Monitoring Agents enrich entity data; Entity context stores and deduplicates |
| BC7 (Policy Engine) | BC8 (Connectors) | Customer-Supplier | Policies may query external data sources during execution |
| BC1 (Identity) | ALL | Open Host | Provides authentication and authorization services to all contexts |
| BC11 (Governance) | ALL | Conformist | All contexts emit audit events; Governance context consumes and stores |
| BC12 (Notifications) | ALL | Open Host | All contexts can publish notifications; Notification context routes delivery |
| BC10 (Growth) | BC5 (Tasks) | Customer-Supplier | Growth tracks onboarding/activation events from Task creation and completion |
| BC10 (Growth) | BC3 (Market Intel) | Customer-Supplier | Published dashboards feed SEO; Growth tracks publishing events |

## 3. Client Side vs. Operating Side Mapping

Most contexts serve both sides with different permission levels:

| Context | Client Side | Operating Side (additional) |
| --- | --- | --- |
| BC1: Identity | User management, workspace switching | All client management, team management |
| BC2: Entity | Entity viewing, entity within workspace | Cross-tenant entity view, global entity admin |
| BC3: Market Intel | Consume dashboards, request research | Create market dashboards, admin Monitoring Agents |
| BC4: Liquidity | Participate in Liquidity Channel | Govern Liquidity Channel, admin triggers, all-client portfolio view |
| BC5: Tasks | Create/manage own Tasks and Channels | Internal Tasks, themed Channels (finance, CS, bugs) |
| BC6: Agent Orchestration | Use and create Team Agents, consume Liquid Agents | Admin all agents, configure agent availability per client |
| BC7: Policy | Create workspace policies, use Policy Builder | Admin default policies, publish policies to clients |
| BC8: Connectors | Use available connectors | Admin connectors, create new connectors, control availability |
| BC9: Billing | View own costs, manage billing | Admin plans, manage client billing, usage overview |
| BC11: Governance | View own audit log | View all audit logs, compliance reporting |

## 4. Domain Events Crossing Context Boundaries

Key events that flow between contexts:

| Event | Emitted By | Consumed By | Significance |
| --- | --- | --- | --- |
| `TaskCreated` | BC5 | BC6, BC12 | Agent Orchestrator picks up new tasks; Inbox notifies |
| `PolicyEvaluated` | BC7 | BC6, BC11 | Agent receives policy result; Audit logs decision |
| `LiquidityTriggered` | BC4 | BC5, BC6, BC12 | Creates Liquidity Channel; Activates agents; Notifies stakeholders |
| `TransactionCompleted` | BC4 | BC9, BC11 | Billing calculates take rate; Audit records transaction |
| `EntityUpdated` | BC2 | BC3, BC4, BC6 | Market Intelligence updates; Liquidity re-evaluates; Agents refresh context |
| `DashboardPublished` | BC3 | BC10 | Growth indexes for SEO |
| `BureauDataReceived` | BC8 | BC6, BC7 | Agents and Policies consume fresh data |
| `RoutineExecuted` | BC5 | BC6, BC3 | Agents perform scheduled work; Intelligence updates |
| `UserOnboarded` | BC1 | BC10, BC12 | Growth tracks activation; Inbox sends welcome |
