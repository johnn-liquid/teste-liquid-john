# 3.3 — Value Flywheel

> **Layer:** 3 — Product Strategy
> 

> **Question this answers:** How does one action generate the next? What are the compounding loops?
> 

> **Last updated:** April 2026
> 

---

## 1. Primary Flywheel: Intelligence → Operations → Liquidity

```
More stakeholders on platform
        ↓
More data generated (queries, analyses, transactions)
        ↓
Better proprietary knowledge base (Monitoring Agents)
        ↓
Higher quality intelligence and risk assessments
        ↓
More trust from market participants
        ↓
More stakeholders attracted (network effect)
        ↓
More liquidity opportunities identified (triggers)
        ↓
More transactions facilitated (take rate revenue)
        ↓
More operational expertise embedded in agents
        ↓
Better platform → attracts more stakeholders
```

**Key insight:** Each revolution of the flywheel makes the next revolution faster. Data compounds, intelligence compounds, trust compounds, network effects compound.

---

## 2. Liquidity Flywheel (Specific)

More data → More precision in risk assessment → More trust from buyers/sellers → More transaction volume → More data about real transactions → Better precision

---

## 3. Acceleration Points

| Point | Why It Accelerates |
| --- | --- |
| **First 5 incorporadoras on platform** | Creates enough transaction volume for meaningful data |
| **Monitoring Agents reach 100+ entities** | Knowledge base becomes genuinely proprietary |
| **First published dashboard indexed by Google** | Organic acquisition begins |
| **First cross-client entity intelligence used** | Demonstrates unique platform value |
| **First liquidity transaction with take rate** | Validates the entire thesis |

## 4. Friction Points

| Point | Why It Slows |
| --- | --- |
| **Onboarding complexity** | Regulated credit platform may be hard to self-serve |
| **Cold start problem** | Intelligence layer is poor without data; data requires users |
| **Compliance overhead per new market** | Each jurisdiction adds regulatory cost |
| **Trust building** | Financial stakeholders move slowly; trust takes time |
| **Token cost at scale** | Agent operations have marginal cost that must be managed |
