# 3.4 — Growth Architecture (Strategy)

> **Layer:** 3 — Product Strategy
> 

> **Question this answers:** How does Liquid grow? Acquisition, expansion, retention, monetization loops.
> 

> **Last updated:** April 2026
> 

---

## 1. Growth Loops

### Acquisition Loop (PLG)

1. Market intelligence dashboards published (SEO-indexed)
2. Stakeholder discovers Liquid via search/social
3. Self-serve registration and exploration
4. Free intelligence layer creates initial value
5. User invites team members / shares dashboards
6. Organic growth within market network

### Expansion Loop

1. Client starts with credit analysis (wedge)
2. Success with credit → adopts repasse management
3. Repasse success → explores portfolio assessment
4. Portfolio assessment → liquidity transaction opportunity identified
5. Liquidity transaction → take rate revenue
6. Success story → referral to other incorporadoras

### Retention Loop

1. Policies configured (switching cost increases)
2. Agents trained on client’s patterns (value increases over time)
3. Audit trail becomes compliance asset (can’t leave without recreating)
4. Monitoring Agents provide ongoing value (continuous, not one-time)
5. Network effects with connected stakeholders

### Monetization Loop

1. SaaS subscription (baseline)
2. More agent actions → more per-action billing
3. Liquidity trigger → take rate revenue
4. Success → plan upgrade → higher subscription

## 2. Product-Led vs. Sales-Assisted

| Motion | When | How | Metric |
| --- | --- | --- | --- |
| **PLG** | Discovery, initial engagement, basic usage | SEO, self-serve onboarding, free intelligence layer, published dashboards | Activation rate, time-to-value |
| **Sales-Assisted** | Enterprise accounts, complex onboarding, liquidity operations | CS-led expansion, dedicated onboarding, strategic partnerships | Expansion revenue, NRR |
| **Community-Led** | Authority building, network effects | Content, events, shared intelligence, stakeholder engagement | Community size, engagement rate |

## 3. Priority Channels

| Channel | Type | Priority | Rationale |
| --- | --- | --- | --- |
| **SEO (published dashboards)** | Organic | P1 | Free acquisition; compounds over time; validates PLG |
| **Direct sales (existing network)** | Outbound | P1 | Soma relationships provide warm leads |
| **Content/social (market intelligence)** | Organic | P2 | Authority building; supports SEO |
| **Referral (client-to-client)** | Viral | P2 | Network effects within incorporadora community |
| **Partnerships (banks, securitizadoras)** | Strategic | P3 | Long cycle but high-value relationships |

## 4. Pricing / Billing Model (Conceptual)

| Tier | Target | Pricing Model | Includes |
| --- | --- | --- | --- |
| **Personal** | Individual professionals | Free / low subscription | Basic intelligence, limited agent interactions |
| **Professional** | Small incorporadoras, brokers | Subscription + usage | Credit analysis, basic repasse, dashboards |
| **Business** | Mid incorporadoras | Higher subscription + usage | Full agent suite, policy builder, channels, team agents |
| **Enterprise** | Large incorporadoras, banks, funds | Custom pricing | Everything + liquidity operations, dedicated support, custom connectors |
| **Liquidity** | Transaction participants | Take rate on transactions | Applied to any liquidity operation facilitated by Liquid |

**Minimum ticket is elevated** (brain dump: "assinatura alta, ticket mínimo elevado"). Liquid is not a low-cost play.

## 5. Expansion Signals

| Signal | What It Indicates | Action |
| --- | --- | --- |
| Client increases agent action volume | Growing adoption | Propose plan upgrade |
| Client creates custom Team Agents | Deep integration | Offer advanced features |
| Client’s portfolio reaches liquidity threshold | Transaction opportunity | Activate Liquidity Agent |
| Client shares dashboards externally | Becoming champion | Leverage for referrals |
| Multiple users from same workspace | Team adoption | Propose Business/Enterprise tier |
